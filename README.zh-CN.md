## 一句话说明

这是一个 **TRX / ETH 靓号地址生成器**。

你输入想要的规则，比如尾号 `8888`，软件会在本机离线随机生成地址，直到找到符合规则的地址。找到后会保存地址、私钥、助记词等你选择保存的内容。

## 两个主要页面

### 1. 靓号生成

用来设置链类型、生成来源、匹配方式、线程数、保存目录，然后开始跑。

![靓号生成页面](./docs/images/generator-page.png)

### 2. 靓号筛选

用来筛选疑似靓号。可以实时同步当前生成出来的疑似结果，也可以导入之前保存的 TXT 文件。

![靓号筛选页面](./docs/images/filter-page.png)

## 生成配置怎么理解

- 链类型：选 TRX 或 ETH。
- 生成来源：选私钥或助记词。
- TXT 保存内容：决定结果文件里保存私钥、助记词，还是两个都保存。
- 匹配方式：决定你要找前缀、后缀、包含，还是组合规则。
- 目标数量：找到几个目标结果后停止，可以填 1、10，也可以填无限。
- CPU 线程数：线程越多通常越快，但电脑也会更忙。
- CPU 加速：提高单次批量生成效率。
- GPU 状态监控：只看显卡状态，不代表 GPU 参与生成。
- 自动保存：建议打开，防止意外关闭丢统计。
- 私钥加密保存：可选，不强制。你自己决定是否加密。

## 大白话难度解释

靓号不是固定进度条。

比如 TRX 后缀 4 位，大概是：

```text
58^4 = 11,316,496
```

也就是说平均大约一千多万次尝试出一个，不是跑到 100% 就必出。

软件显示的是概率进度：

```text
累计命中概率 = 1 - (1 - p)^n
```

简单说就是：你尝试得越多，命中概率越高，但随机事件没有绝对保证。

## 疑似靓号规则

疑似靓号只看地址后缀。

支持：

```text
豹子号：8888 / 9999 / aaaa / 7777777 / 88888888
数字顺子：12345 / 234567 / 98765 / 876543
字母顺子：abcde / edcba
自定义后缀：你自己填写
```

不支持自动识别一堆英文单词。你要英文后缀，就自己填到自定义后缀里。

## 筛选功能怎么用

筛选页有两种模式：

```text
实时同步：边生成边把疑似靓号送到筛选页
导入 TXT：选择之前保存的 suspicious.txt 进行筛选
```

筛选结果会显示：

- 链类型
- 地址
- 分类
- 私钥复制按钮
- 助记词复制按钮
- 地址复制按钮

私钥和助记词太长，所以不会整段铺在表格里，避免看起来乱。

## 文件保存

目标靓号：

```text
results/results.txt
```

疑似靓号：

```text
results/suspicious/suspicious.txt
```

每行大概长这样：

```text
地址 私钥
地址 私钥 助记词
```

具体保存什么，看你在界面里勾选了什么。

## 本地离线说明

程序源码里做了离线扫描：

```bash
npm run verify:offline
```

当前源码不包含直接联网 API。也就是说，它不是那种把私钥发到服务器上的工具。

但是还是那句话：你自己的电脑环境要安全。如果电脑本身中了木马，那任何本地钱包工具都有风险。

## 验证命令

```bash
npm install
npm test
npm run verify:offline
```

测试会校验：

- ETH 私钥和地址是否匹配
- TRX 私钥和地址是否匹配
- 私钥格式是否能用于钱包导入
- 助记词模式是否正常
- 加密保存和错误密码是否正常
- checkpoint 是否能恢复统计
- 疑似靓号是否只按后缀识别

## 安全提醒

私钥和助记词不是普通文本，它们就是钱包控制权。

请记住：

- 不要把私钥发给别人。
- 不要把助记词发给别人。
- 不要把带私钥的截图发给别人。
- 不要在不安全的电脑上生成大额钱包。
- 正式使用前先用空钱包测试导入。
