# 中文详情

这是一个本地离线桌面版 **TRX / ETH 靓号地址生成器**。它面向需要自己生成地址、自己保管私钥的用户，重点是本地、安全、可验证。

[返回首页](./README.md)

## 当前版本说明

- 当前版本使用 **CPU 多线程** 生成地址。
- GPU 只做状态监控，不参与地址生成。
- 疑似靓号只检测地址后缀。
- ETH 不再内置 `dead / beef / cafe / face / feed`，需要英文后缀时请在自定义后缀里手动填写。

## 生成配置

支持：

- 链类型：TRX / ETH
- 生成来源：私钥 / 助记词
- TXT 保存内容：私钥 / 助记词 / 两者都保存
- 匹配方式：前缀、后缀、包含、组合规则
- 目标数量：指定数量或无限
- CPU 线程数
- 每批生成数量
- 自动保存
- 可选私钥加密保存

## 疑似靓号规则

疑似靓号只看后缀：

```text
Txxxxxx8888  -> 命中
T8888xxxxxx  -> 不命中
Txxx8888xxx  -> 不命中
```

支持三类：

```text
后缀豹子号：8888 / 9999 / aaaa
后缀顺子号：12345 / 98765 / abcde / edcba
自定义后缀：用户自己输入，一行一个
```

默认：

```text
豹子最少位：4
顺子最少位：5
每批生成：512
```

## 保存逻辑

目标命中结果保存到：

```text
results.txt
```

疑似后缀结果保存到：

```text
suspicious/suspicious.txt
```

默认每行保存：

```text
地址 私钥
```

如果选择助记词保存，会追加助记词。

## 验证

运行测试：

```bash
npm test
```

离线扫描：

```bash
npm run verify:offline
```

CPU worker 基准：

```bash
node scripts/benchmark-workers.js 10 TRX 5
```

## 风险提示

生成出来的私钥和助记词就是实际钱包资产凭证。请不要把结果文件、截图、私钥、助记词发给任何人。正式使用前，建议先用空钱包导入验证。
